UI Layer -- Presentation
CcDirector.Wpf
Primary Windows desktop app. WPF with XAML data binding.
  • MainWindow -- 3-panel layout
  • SessionViewModel -- ObservableCollection
  • NewSessionDialog, LoadWorkspaceDialog
  • EmbeddedConsoleHost (native overlay)
  • Voice recording UI
CcDirector.Avalonia
Cross-platform UI (in progress). Same layout, .axaml format.
  • Windows + macOS + Linux
  • CommunityToolkit.Mvvm
  • Shared Core backend
CcDirector.Terminal
Custom ANSI terminal renderer for WPF.
  • DrawingVisual-based rendering
  • 50ms poll loop from buffer
  • Colors, bold, italic, selection
CcDirector.Terminal.Avalonia
Cross-platform terminal renderer for Avalonia.
Core Services -- Business Logic
SessionManager
Manages all active Claude sessions.
  • ConcurrentDictionary<Guid, Session>
  • CreateSession / CreateEmbeddedSession
  • Session resumption via --resume
Session
Single Claude Code instance + state machine.
  • ActivityState: Starting -> Idle -> Working -> WaitingForInput
  • SendText / KillAsync / HandlePipeEvent
  • Terminal buffer, process ID, metadata
Configuration
Settings, repos, persistence.
  • RepositoryRegistry (repositories.json)
  • SessionStateStore / RecentSessionStore
  • AgentOptions from appsettings.json
  • RootDirectoryStore
Claude Integration
Claude Code session discovery and metadata.
  • ClaudeSessionReader -- .jsonl parsing
  • ClaudeSessionMetadata
  • McpConfigManager
  • AgentTemplateStore
  • UsageHistoryStore
CcDirector.Engine
Background jobs, dispatch, archival.
  • Dispatcher -- email routing
  • Archival -- vault backup
  • Jobs -- background tasks
Terminal Backends -- Process Management
ConPtyBackend
Windows Pseudo-Console API.
  • PseudoConsole + ProcessHost
  • Async I/O pipes
  • CircularTerminalBuffer (2 MB)
UnixPtyBackend
Mac/Linux PTY via openpty().
  • UnixPseudoConsole
  • TIOCSWINSZ resize
EmbeddedBackend
Native console window overlay (WPF only).
  • SetParent() window reparenting
  • WriteConsoleInput()
PipeBackend / StudioBackend
Stateless prompt mode + stream-JSON mode.
  • -p --output-format stream-json
  • Card-based UI rendering
IPC / Hook System -- Control Plane
HookInstaller
Installs 14 hook types into Claude's settings.json.
  • SessionStart, UserPromptSubmit
  • PreToolUse, PostToolUse
  • Notification, Stop, SubagentStart...
Hook Relay
PowerShell (Win) / Python (Unix) relay scripts.
  • Reads JSON from Claude's stdin
  • Sends to named pipe / Unix socket
EventRouter
Routes PipeMessages to correct Session by session_id.
  • Deduplication
  • Session ID -> Guid mapping
  • ActivityState transitions
DirectorServer
IPC listener (named pipe on Win, Unix socket on Mac/Linux).
External Systems
Claude Code CLI
claude.exe -- the AI coding assistant process.
  • Spawned per session
  • --session-id, --resume flags
  • 14 hook event types
Git
GitStatusProvider polls repo status.
MCP Servers
Model Context Protocol servers managed via Claude config.
Vault (SQLite)
Contacts, tasks, goals via cc-vault CLI.

Data Flow -- Session Lifecycle

  [User clicks "New Session"]
          |
          v
  MainWindow.xaml.cs -----------> NewSessionDialog (repo picker)
          |                                    |
          | repoPath selected                  |
          v                                    v
  SessionManager.CreateSession(repoPath, args)
          |
          |--- Creates Session with Guid, adds to ConcurrentDictionary
          |--- Sets env var CC_SESSION_ID
          |
          v
  ISessionBackend.Start()
          |
          +---> ConPtyBackend (Windows)  -- spawns claude.exe via PseudoConsole
          +---> UnixPtyBackend (Mac)     -- spawns via openpty()
          +---> EmbeddedBackend (WPF)    -- native console overlay
          |
          v
  claude.exe starts with hooks installed
          |
          |--- Data plane:  terminal bytes --> CircularTerminalBuffer --> TerminalControl (50ms poll)
          |
          |--- Control plane: hook events --> hook-relay.ps1 --> Named Pipe
          |                                                              |
          v                                                              v
  UI shows live terminal                              EventRouter.Route(PipeMessage)
                                                                         |
                                                                         v
                                                              Session.HandlePipeEvent()
                                                                         |
                                                                         v
                                                              ActivityState transition
                                                              (Starting -> Idle -> Working -> WaitingForInput)
                                                                         |
                                                                         v
                                                              UI color/badge updates via data binding

Dual-Channel Architecture

  CC Director uses two independent channels per session:

  DATA PLANE (Terminal)                     CONTROL PLANE (Hooks)
  ========================                  ========================

  claude.exe                                claude.exe
       |                                         |
       | raw bytes (stdout)                      | hook events (JSON)
       v                                         v
  ConPTY / UnixPTY                        hook-relay.ps1
       |                                         |
       v                                         v
  CircularTerminalBuffer                  Named Pipe / Unix Socket
  (2 MB ring buffer)                           |
       |                                         v
       | 50ms poll                            DirectorFileEventWatcher
       v                                         |
  TerminalControl                              v
  (ANSI parser -> cells)                  EventRouter
       |                                         |
       v                                         v
  User sees terminal                     Session.ActivityState
                                                  |
                                                  v
                                            UI badges + colors
