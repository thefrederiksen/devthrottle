Issue #862 - Flaky telemetry tests

Developer Agent proof. Branch issue-862-flaky-telemetry (off main).

I believe this is finished.

1. Corrected root cause

The issue first guessed test parallelism. On reading the code that proved wrong: CcDirector.Gateway.Tests already disables parallelization (TestParallelization.cs -> [assembly: CollectionBehavior(DisableTestParallelization = true)]), so the tests run serially. The real cause is the single, process-wide FileLog background writer:

  • Carryover - the shared writer is never stopped between serial tests, so a previous test's still-queued lines flush into the current test's file after it snapshots/clears the shared log directory; a "never writes the token" scan can then see another test's line.
  • Flush timing - FileLog flushes on a background thread (1-second interval / on queue-drain). The tests waited with a fixed Task.Delay / a poll loop; under CI load the expected line had not landed in time, so the "records locally" / "logging is wired" assertions failed.

2. The fix

  • A test-only seam on FileLog (src/CcDirector.Core/Utilities/FileLog.cs): RedirectForTests() returns a scope that swaps the single static writer for a fresh one pointed at a private, throwaway directory, and DrainAndReadLines() synchronously drains it (Stop() completes the queue and joins the writer thread) and returns exactly the lines that scope produced. Restored + the temp dir deleted on dispose. Internal (Core already grants InternalsVisibleTo CcDirector.Gateway.Tests); safe because the test assembly is serial, so one test owns FileLog at a time. No production behavior change - the live logger is untouched.
  • The three tests now use the seam instead of snapshotting/clearing the shared CcStorage.ToolLogs("director") directory: no baseline bookkeeping, no Task.Delay, no poll loops. Each reads a private, fully-flushed log - deterministic by construction. (Dead ReadAllLinesShared helpers removed.)
