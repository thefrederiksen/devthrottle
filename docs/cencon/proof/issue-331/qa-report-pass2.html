Context: What Was Fixed from Pass 1

Pass 1 QA found that BuildLauncherClient in MachineEndpoints.cs hardcoded http://127.0.0.1:<port>/ for all relay calls, making cross-machine relay physically impossible. LauncherRegistrationRequest and LauncherRegistry had no network address field.

The fix commit (1d43e32) adds:

  • NetworkAddress to LauncherRegistrationRequest and LauncherDto (contracts)
  • LauncherRegistry.GetNetworkAddress() - exposes the stored tailnet address
  • BuildLauncherClient(port, token, networkAddress) - uses <networkAddress> when non-empty, falls back to 127.0.0.1 when empty (co-located launcher)
  • GatewayRegistrationClient sends Environment.MachineName as NetworkAddress on registration
  • 6 new tests proving the address flows end-to-end
