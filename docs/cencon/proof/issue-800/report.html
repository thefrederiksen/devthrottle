Issue #800 - Enforce meaningful session names at creation (name/purpose, never bare folder)

Developer Agent proof. Branch issue-800-session-names. Build: dotnet build cc-director.sln -> Build succeeded. 0 Warning(s), 0 Error(s). Live proof captured against an isolated test Director (slot 14, Control API http://127.0.0.1:7880), launched via its own scheduled task per CLAUDE.md rule 0b.

What was implemented

  • Single name composer - new SessionName in CcDirector.Core/Sessions/SessionName.cs: Compose(folder, type, explicitName, purpose, disambiguator), IsWeakExplicitName, DisplayName, FolderName, Disambiguator. It is the one place the naming rule lives.
  • Contract - NewSessionRequest gains optional Name and Purpose fields (CcDirector.Gateway.Contracts).
  • Name at birth - SessionManager.CreateSession takes an optional Func<Guid,string> nameFactory; the new session's CustomName is set in the create call, not by a later rename. The id is passed so the name can carry an id-derived disambiguator (no new persisted state).
  • POST /sessions - composes the final name through the composer and enforces the rule (below), setting CustomName on the new session before returning 201.
  • De-duplicated display fallbacks - the bare-folder fallbacks at ControlEndpoints.cs ~231 and ~256, SessionManager.cs ~322 (plus the same pattern in the voice endpoint ~1538) now route through SessionName.DisplayName.
  • cc-devthrottle - session spawn gains --purpose (forwarded to the new Purpose field), keeps --name, and prints a warning when NEITHER is supplied.
  • Docs - the fleet-comms skill spawn examples lead with --name/--purpose and state the always-name-your-session norm.
  • Tests - SessionNameTests (15 cases) cover passthrough, purpose composition, type+disambiguator default, weak-name rejection, and disambiguator-distinctness.
Enforcement rule (the agreed middle path): an EXPLICIT name (req.Name supplied) that is blank or equals (case-insensitive) the bare folder name is rejected with HTTP 400. An ABSENT name (req.Name == null) is auto-composed from folder + purpose, else folder + type + disambiguator, so a session can never display as the bare folder name alone.
