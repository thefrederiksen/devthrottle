What Was Implemented

A new POST /sessions/{id}/voice-turn endpoint that encapsulates the full server-side voice turn loop. The client sends either raw audio bytes or pre-transcribed text. The server:

  1. Transcribes audio (via Whisper) if audio was provided
  2. Waits until the session is ready (emits waiting events while polling)
  3. Posts the text to the session via SendTextAsync
  4. Polls until the turn completes (emits thinking events while polling)
  5. Calls ClaudeSummarizer to produce 2-3 sentences of plain spoken prose
  6. Synthesizes the summary to speech via TtsService
  7. Emits the final reply event with summary + audioBase64

If the wingman or TTS is unavailable, a fallback path produces a plain-text excerpt and empty audio. The reply event is always emitted - never goes silent.
