Method note on the visual screenshots

The QA session ran from a non-interactive automation context where the GDI screen device context was unavailable, so direct pixel capture (CopyFromScreen) reported "handle is invalid". The QA session therefore proved which window appeared with two stronger, automation-safe signals: the operating system window title reported by the process (Process.MainWindowTitle) and the window rectangle. The account gate screen has Title="Sign in to DevThrottle" and a 520x420 client; the main window has Title="CC Director". These titles unambiguously distinguish the two windows. The committed PNGs (developer artifacts) show the same windows the QA titles confirm.
