Issue #593 - Retire the dormant Vosk offline path

Branch: issue-593-retire-vosk   |   Parent: #585   |   Design gap: G8

Release Notes

The dormant offline Vosk speech-to-text path (CcDirector.VoskStt) has been retired so the codebase carries exactly one transcription stack: the shared cloud batch pipeline plus the validated dictionary corrector. The Vosk project was never wired into any live transcription path (desktop, voice, and recordings all use the shared pipeline), and it carried a second, divergent dictionary implementation (a Soundex find-and-replace) separate from the live corrector. Both are now parked.

Decision: parked (reversible), not hard-deleted

The issue's own Assumptions say "Confirm before deleting versus parking," and no human confirmed a hard delete. Following the repository's established convention for retired projects (archived/CcDirector.Wpf, archived/CcDirector.CommunicationManager, both with status: archived in the manifest), the Vosk source was moved (git rename, fully reversible) to archived/CcDirector.VoskStt and dropped from cc-director.sln. No source was deleted. This satisfies every acceptance criterion: no live code references the Vosk path, the solution builds clean without it, and the manifest is updated.

Changes

  1. archived/CcDirector.VoskStt/ - the entire project (CcDirector.VoskStt.csproj, VoskSttService.cs, CustomDictionary.cs) moved here from src/ via git mv (git records a pure rename - reversible).
  2. cc-director.sln - removed the CcDirector.VoskStt Project/EndProject block, its 12 build-configuration lines, and its NestedProjects entry. The project no longer participates in the build.
  3. docs/cencon/architecture_manifest.yaml - the vosk_stt container entry now has status: archived, project: archived/CcDirector.VoskStt, and a description noting it is not part of the live pipeline. last_updated annotated.
