Issue #659 - QA Agent independent verification

Independent QA verification of the installer Privacy step (three plain-English one-liners plus one default-on per-account telemetry toggle). The QA Agent did not trust the Developer Agent's report or screenshots: it reviewed the actual source on the pull request branch, ran the unit tests itself, and exercised the real production HTTP client and the real local config.json mirror against a live local stub server it stood up itself.

How this was verified independently

  • Source review on the PR branch (issue-659-privacy-step): the Privacy step XAML was checked line by line against the spec (the three exact one-liners and the exact checkbox text "Share anonymous usage information to help improve CC Director", with IsChecked="True" as the default). The pre-fill / hint logic in the code-behind and the best-effort, never-blocking apply wiring in MainWindow were read in full.
  • Unit tests run by QA (not reused from the developer): AccountTelemetryClientTests passed 9/9; TelemetryChoiceApplierTests passed 8/8.
  • Live-stub round trip run by QA: a self-built console harness pointed the real AccountTelemetryClient at a live HttpListener stub via DEVTHROTTLE_API_URL, with CC_DIRECTOR_ROOT isolating config.json to a throwaway temp directory. It captured every request on the wire and read back the local mirror. Transcript: qa-live-stub-transcript.txt in this folder.
  • Token-never-logged audit: a search across the changed source confirmed no log statement writes the access-token value; the token is held in memory only and is never placed on the sign-in result.

The real production sign-in is not live, and a development stand-in token is not a real Supabase JWT, so the production consent endpoint returns 401. Expected: against a real account, the same call shape writes the per-account server flag. Actual (accepted): the call shape (method, URL, Bearer header, body) and the local config.json mirror were verified against a local stub implementing the devthrottle_internal#59 contract; production verification is deferred until real sign-in is live. This is the documented, accepted limitation in the issue and does not fail the issue.
