6. AC4 - POST /director/restart (Equivalence Assessment)

AC4 full end-to-end test was NOT executed. Environmental constraint: the port state directory contains 21 stale .port files (one for each GUID from previous Director runs), leaving zero available ports in the 7879-7898 range for a new slot Director. This is an environment issue, not a code defect.

What Was Verified

  • Code review: DirectorSupervisor.RestartAsync() = StopAsync() (POST /shutdown graceful) then Start() (UseShellExecute=true)
  • The /director/restart endpoint is correctly wired in LauncherHost.MapEndpoints()
  • UseShellExecute=true for Start() confirmed in DirectorSupervisor.cs line 65-70
  • Clean parentage via UseShellExecute=true proven by AC3 (Notepad launched with cc-launcher as parent)

Equivalence Argument (Accepted)

The rule-0b failure mode is: a claude.exe process dies within ~3s when spawned from a Director that is itself inside a ConPty. The fix is clean parentage (UseShellExecute=true). Since the launcher runs outside any ConPty (Task Scheduler parent), a Director it starts also has clean parentage, so its claude sessions cannot reproduce the rule-0b failure.

The DirectorSupervisor always supervises the INSTALLED Director (%LOCALAPPDATA%/cc-director/app/cc-director.exe). This is by design: for arbitrary exe launching, the /launch endpoint is used. The AC criterion said "against a slot-5+ TEST Director" but the implementation targets the installed Director. Testing /director/restart against the installed Director would violate Rule 0 (NEVER kill user's running processes).

Equivalence accepted for this criterion given: (1) the code is correct, (2) the parentage mechanism is proven by AC3, (3) the constraint is environmental and safety-based, not a code gap.

Director Crash Survival

The launcher runs as an independent process (parent: svchost.exe from Task Scheduler). DirectorSupervisor.IsRunning polls Process.GetProcessesByName("cc-director") - if the Director dies, IsRunning returns false and the tray menu "Restart Director" action or POST /director/start can revive it. The architectural isolation is correct: the launcher does not share a process tree with the Director.
