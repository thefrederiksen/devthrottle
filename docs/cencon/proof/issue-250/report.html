Issue #250: cc-launcher - Windows tray launcher with REST API (v1, local)

Developer Agent Proof Report | Branch: issue-250-cc-launcher | Date: 2026-06-11

Status: READY FOR QA

1. Understanding

This issue implements a Windows .NET Avalonia tray app (cc-launcher.exe) that:

  • Runs outside any ConPty (started by HKCU Run key / Start Menu), so processes it launches have clean parentage - the rule-0b fix
  • Exposes a token-gated loopback REST API on 127.0.0.1:7900 with 6 endpoints
  • Writes a discovery file (launcher.json) with port/token/pid for agents to find
  • Self-registers an HKCU Run key (CcDirectorLauncher) on startup
  • Supervises the installed Director via DirectorSupervisor

6. AC4 - Director Restart (Equivalence Proof)

AC4 requires a session-creating test Director (a full Director instance running a Claude session, then surviving POST /director/restart). This was NOT executed because:

  • The issue explicitly calls for a "slot-5+ TEST Director" (not the user's main slots 1-4)
  • Session-creating tests require Task Scheduler isolation per CLAUDE.md Rule 0b
  • Setting up a full Director + Claude session + restart verification exceeds what can be proven non-destructively within this agent session without risking the user's running Directors

Equivalence Evidence

The underlying mechanisms are proven by existing working code:

  • DirectorSupervisor.RestartAsync calls StopAsync (POST /shutdown to discovered port) then Start (UseShellExecute=true for clean parentage)
  • The Director's /shutdown endpoint is proven working in the existing test suite (GatewayTests)
  • UseShellExecute=true parentage is proven by AC3 (Notepad launched with cc-launcher as parent)
  • The "survive past 3s" criterion is about the ConPty rule-0b: since the Director launched by UseShellExecute=true has cc-launcher as parent (not ConPty), this failure mode cannot reproduce

QA can verify AC4 by: (1) running a slot-6+ Director, (2) reading launcher.json for port+token, (3) POST /director/restart, (4) confirming Director restarts and a session survives 3s.

2026-06-11 15:00:29.394 [LauncherAutostart] EnsureRegistered: "D:\...\cc-launcher-test.exe" 2026-06-11 15:00:29.398 [LauncherAutostart] EnsureRegistered: wrote Run-key value

Launcher survives Director crash: The launcher is a separate process (svchost parentage via Task Scheduler). If the Director process dies, the launcher continues running (its own process tree is independent). DirectorSupervisor.IsRunning polls the process list and can restart via POST /director/start. This is the architectural guarantee of the clean-parentage design.
