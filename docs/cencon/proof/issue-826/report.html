Branch: issue-826-tool-reconciler   |   Scope: setup engine + tests only (no UI, no App.axaml.cs lifecycle, no settings/config key - those are follow-ups #827/#828/#829).

What was implemented

  • New: tools/cc-director-setup-engine/ToolReconciler.cs - the ToolReconciler.ReconcileAsync(...) operation, the ReconcileResult record, and the ReconcileOutcome enum (InSync / Reconciled / Failed). It is the automatic, idempotent, lightweight equivalent of the manual "Fix Tools" repair.
  • Edited: tools/cc-director-setup-engine/PythonToolsInstaller.cs - exposed existing reuse points without duplicating logic: made RemoveLegacyAliasShims() and WriteShims(...) public, and added FindOrphanedLegacyAliasShims() (pure detection) sharing one private path-set helper with the purge.
  • New: tools/cc-director-setup-engine.Tests/ToolReconcilerTests.cs - 8 offline tests (temp/fake layout, injected fake heavy-repair, no network).

How it works (reusing existing detection, not a parallel comparison)

  • (a) Missing shim: enumerates the embedded manifest via ToolCatalogService.GetCatalog(); for a tool whose venv console-script exe exists (PythonToolsInstaller.ConsoleScriptPath) but whose bin\<name>.cmd shim is absent, it creates the shim directly via PythonToolsInstaller.WriteShims.
  • (b) Orphaned legacy shims: detected with PythonToolsInstaller.FindOrphanedLegacyAliasShims() (reusing LegacyAliasShimNames) and purged with RemoveLegacyAliasShims() - the legacy list is never duplicated.
  • (c) Broken/absent venv: detected with PythonToolsState.LoadScripts + PythonToolsInstaller.VenvHasAllTools; only this escalates to the heavy release-backed ToolUpdater.RepairPythonToolsAsync.
  • Cheap happy path: drift is fully computed from reads first; with no drift the method performs zero filesystem mutation, never calls the heavy repair, and returns InSync.
  • Mutex: the heavy escalation is guarded by the machine-wide named mutex Global\cc-director-tool-reconcile (same convention as TailscaleCli.ServeMutexName / agent-session-isolation.ps1). If another Director holds it, the call logs "reconcile skipped - another Director is reconciling" and returns without forcing. The light shim path does not take the heavy lock.
