Issue #392 - [Desktop] First-run tool-detection wizard

Developer Agent proof. A first-run wizard auto-opens on the Director desktop when no agent tools are configured, scans for known catalog tools via the existing ToolDetectionService, auto-suggests found tools pre-filled with the recommended Standard command line + default model (issue #391's catalog), lets the user deselect before accepting, writes accepted tools to the machine-level config.json, can be re-run from the Agents tab in Settings, does not re-trigger once configured, and never auto-adds --dangerously-skip-permissions.

What was implemented

  • CcDirector.Core/Agents/ToolDetectionWizardModel.cs (new) - UI-free engine: IsFirstRun() (true when agent.tools is absent/empty in config.json), ScanSuggestions() (one suggestion per catalog tool, reusing ToolDetectionService.DetectTool), and AcceptSelected() (writes the selected tools via AgentToolConfig.FromCatalogDefaults().Save() plus the detected path under agent.<key>_path; never skip-permissions).
  • CcDirector.Avalonia/ToolDetectionWizardDialog.axaml(.cs) (new) - the wizard window: per-tool card with a found/not-found badge, a pre-checked checkbox for found tools, and the recommended command line + model. Async scan, responsive UI, Add selected tools / Skip for now.
  • CcDirector.Avalonia/MainWindow.axaml.cs - MaybeShowToolDetectionWizard() at the end of MainWindow_Loaded auto-opens the wizard on first run only.
  • CcDirector.Avalonia/SettingsDialog.axaml(.cs) - a "Detection wizard..." button on the Agents tab re-runs the wizard on demand and reloads the page after accept.
  • CcDirector.Core.Tests/Agents/ToolDetectionWizardModelTests.cs (new) - 7 unit tests.
