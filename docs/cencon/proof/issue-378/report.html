Proof - Issue #378: [Voice] Phone app: swap voice-turn URLs from Director to Gateway

Developer Agent | 2026-06-12 | Branch issue-378-phone-gateway-urls | PR #383 | Depends on #376 (merged, commit ead9299)

What was implemented

The phone walkie-talkie agent turn now submits and polls voice turns on the Gateway (POST /sessions/{sid}/voice-turn/submit + GET /sessions/{sid}/voice-turn/{turnId}, added by #376) instead of addressing the owning Director directly. The Gateway base URL comes from the phone's own settings (the gateway_url preference used by every page for the roster), so the DoR question in the issue resolves to: no session-DTO change needed.

Porting note (recorded for QA honesty): the SubmitVoiceTurnAsync/PollVoiceTurnAsync methods the issue names existed only on the unmerged branch issue-347-voice-dictation-recording-indicator (commit 044e1c3 - the build the deployed phone currently runs). Main never had them. This change ports that submit/poll design into main's evolved phone code and points it at the Gateway, per docs/architecture/gateway/VOICE_TURN_ARCHITECTURE.md Phase 3.
