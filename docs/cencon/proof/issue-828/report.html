Issue #828 - Tools settings page: "Automatically keep cc-* tools up to date" opt-out toggle

Developer Agent proof. A user-facing toggle was added to the Tools tab of the Director Settings dialog, bound to the tools.autoUpdate.enabled config key (added in #827 and honored by the lifecycle reconciler). ON by default; turning it off stops all automatic reconciles and leaves the manual "Download and repair tools" button as the only path.

What was implemented

  • src/CcDirector.Core/Configuration/ToolAutoUpdateSetting.cs - new Core read/write path for tools.autoUpdate.enabled. Get() reads via CcDirectorConfigService.ReadRaw() (absent/non-bool falls back to enabled, matching the setup engine's read path so the toggle and the lifecycle never disagree). Set(bool) persists via MergePatch, preserving every other section.
  • src/CcDirector.Avalonia/SettingsDialog.axaml - the toggle, helper text, and a live on/off status line, placed in the Tools card under the repair button. Reuses the dialog's existing style tokens (Classes="hint", inline hex) per VisualStyle.md.
  • src/CcDirector.Avalonia/SettingsDialog.axaml.cs - loads the value off the UI thread, suppresses the initial programmatic set, and persists immediately on toggle via Task.Run(ToolAutoUpdateSetting.Set) with FileLog entries. The status line updates instantly for responsive-UI feedback.
  • src/CcDirector.Core.Tests/Configuration/ToolAutoUpdateSettingTests.cs - 7 view-model-level tests: default ON, missing section/key -> ON, non-bool -> ON, explicit false -> OFF, set persists and survives re-read, round-trips both ways, preserves unrelated sections.
