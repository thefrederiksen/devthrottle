Title: [Gateway] GET /account/status (is the Gateway signed in, and who as)

Branch: issue-638-account-status   |   Affected containers: CcDirector.Gateway, CcDirector.Gateway.Tests

What was implemented

  • New endpoint GET /account/status on the Gateway (src/CcDirector.Gateway/Api/AccountStatusEndpoint.cs), returning { "signedIn": bool, "email"?: string, "provider"?: string }.
  • The status is computed entirely locally from the Gateway-hosted credential service (issue #636, GatewayHost.Account = the reused DevThrottleAccountService): IsLoggedIn() for the boolean and GetIdentity() (backed by JwtIdentityReader) for the email/provider. There is no cloud or network call.
  • Wired into GatewayHost.StartAsync after the telemetry endpoints, so it inherits the existing host-wide Gateway token middleware (AuthMiddleware) exactly like the other Gateway endpoints.
  • The response carries no token field - the response record has only the boolean + identity, so the access/refresh token can never be returned (security rule DT-05). The identity fields are omitted (not emitted as null) when not signed in.
  • Unit tests: src/CcDirector.Gateway.Tests/Account/AccountStatusEndpointTests.cs (5 tests) cover signed-in, not-signed-in, no-credential-service, auth-enabled unauthorized (401), and auth-enabled authorized (200).
