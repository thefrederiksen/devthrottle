[Director] Add a read-only ACCOUNT status box to the sidebar status stack

Branch issue-852-account-box · built as cc-director15.exe (v0.9.29, commit b32c3a2) · isolated slot 15, Control API 127.0.0.1:7879 · captured 2026-06-30.

What was implemented

  • New ACCOUNT status Border in the bottom-left sidebar status stack (src/CcDirector.Avalonia/MainWindow.axaml), declared directly after the GatewayIndicator so the DockPanel.Dock="Bottom" ordering places it above the GATEWAY box. Styled to match the existing Gateway / Tools / Update / Control API boxes (same margin, padding, corner radius, palette).
  • Asynchronous status read on a 30-second heartbeat poll (MainWindow.axaml.cs): the read runs off the UI thread via Task.Run over the existing GatewayAccountStatusClient (GET {gateway.url}/account/status, shipped in #638/#651), then marshals back with Dispatcher.UIThread.Post. The box paints its muted "reading account status..." state immediately and never blocks startup.
  • Three visual states decided by a new pure, unit-tested mapper CcDirector.Core.Account.AccountIndicatorPresenter: SignedIn (green, email), NotSignedIn (amber nudge), Unavailable (muted). The mapper guarantees a not-configured or unreachable Gateway maps to Unavailable and never a false "signed out".
  • Click handler opens the Cockpit Account page only in the not-signed-in state - it asks the configured Gateway for the Tailscale front-door URL (GET {base}/cockpit) and opens {frontDoor}/account, mirroring the existing Cockpit/Learn buttons. The Director never opens a localhost URL.

Not a re-added sign-in gate (#651 / #664)

This box is a read-only, non-blocking nudge. Issues #651/#664 removed the Director's own account gate and require the Director to always boot straight to the main window regardless of sign-in. This change adds no startup gate, block, or sign-in form: it only mirrors /account/status for display and links out to the Gateway-managed Cockpit Account page. The status read is fire-and-forget after the window is shown (proven by the startup timestamps below), so a signed-out, unreachable, or unconfigured Gateway never affects boot.

CenCon impact

No drift. This is an additive, read-only UI box on the already-documented Avalonia desktop surface, reusing the already-documented GatewayAccountStatusClient and the existing Gateway GET /account/status endpoint. No new container, external system, security boundary (same Gateway auth), or pattern. Security rule DT-05 is honored - the box only ever shows the identity (email/provider); it can never surface a token, and the email (PII) is never written to the log.
