Defect (AC1)

Criterion: "With the window at its default size on a 1920x1080 display, opening Settings -> Agents shows at least 3 full agent-tool cards without scrolling." The issue's Assumptions explicitly note "the binding criterion is the 3-card rule."

Expected: 3 complete tool cards (e.g. Claude Code, Pi, Codex) fully visible without scrolling.

Actual: Only 2 full cards (Claude Code, Pi) are fully visible at the default 1040x860 size on 1920x1080. The 3rd card (Codex) is clipped to just its header/Enabled row, and a vertical scrollbar is present in the Agents content area - the user must scroll to see the full 3rd card.

Measurement (UI Automation bounding rectangles, dialog bottom = 1024 px):

Note the Save and Cancel button bar occupies the lower ~60 px of the 860-px dialog, further reducing the usable card area. The expanded hint paragraph (lengthened for AC10) plus the toolbar (Detection wizard / Detect All / Test All) consume enough vertical space that the 3rd card does not fit at 860 px height.

Repro: Open the app on a 1920x1080 display -> Settings -> Agents tab. Observe only Claude Code and Pi are fully visible; Codex is cut off and the scrollbar is active.

Suggested fix direction (developer's call): increase the default dialog Height (and/or reduce vertical chrome: tighten the hint paragraph spacing or the toolbar) until 3 full cards fit at 1920x1080. The Width is fine; the Height (860) is the binding constraint.

Evidence: see ac5-quick-check.png and ac1-ac9-agents-standard-preserved.png (Claude + Pi full, Codex header only, scrollbar visible at right).
