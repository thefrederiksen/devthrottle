Context: the prior bounce

The previous QA pass FAILed this issue because DeviceRegistryClient.RegisterAsync parsed device_key and the record from the JSON root, but the merged cloud returns them enveloped under data. The stub fixtures used the flat (buggy) shape, so the tests were green while masking a deterministic production parse failure. The required fix was: unwrap data.device_key and data.record in RegisterAsync, unwrap data in ListDevicesAsync, and correct the stub fixtures to the enveloped shape so they would actually catch the mismatch.
