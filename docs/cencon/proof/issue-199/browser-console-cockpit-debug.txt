Browser console capture (Edge), with localStorage 'cockpit.debug' = 1, isolated Cockpit on
http://127.0.0.1:7473 (against the live Gateway roster). These lines are emitted by
wwwroot/js/cockpit-terminal.js ONLY when the cockpit.debug flag is set; with the flag absent
the console stays quiet (issue #199 scope C requirement). Each line is prefixed [cockpit-terminal].

In this isolated harness the standalone Cockpit has no Gateway WS proxy on its own origin
(7473), so the per-session stream WebSocket closes with code 1006 and the bounded reconnect
loop fires - which is exactly the remote-WS failure shape #199 wants visible in the console.

Notes on the additional logged events (also gated behind cockpit.debug=1, see cockpit-terminal.js):
- "ws open <url> (<n>ms to open)" - logged on a successful WebSocket open.
- "first frame <n bytes/chars> (<n>ms since connect)" - logged on the first stream frame.
- "gave up after 30 attempts; last close code <n>" - logged when the bounded retry budget is spent.
These three did not appear in THIS capture because the isolated origin has no WS proxy, so the
socket never opened and never reached the give-up threshold during the capture window.
