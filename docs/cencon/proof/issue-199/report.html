Issue #199 - [Cockpit] Persisted INFO logging of user actions + stream/circuit lifecycle

Developer Agent proof · PR branch issue-199-cockpit-logging · verified 2026-06-10

What was implemented

  • A. File sink — new src/CcDirector.Cockpit/Logging/CockpitFileLog.cs: an ILoggerProvider + background writer that routes every Cockpit ILogger line to %LOCALAPPDATA%\cc-director\logs\cockpit\cockpit-YYYY-MM-DD-<PID>.log, matching the Director/Gateway FileLog timestamp format, filename shape and day-rollover/bounded-flush robustness (issue #171 contract). Registered in Program.cs; the configured CcDirector.Cockpit: Debug level still applies (the factory filters before the sink). Self-contained because the lean Cockpit deliberately does not reference Core (Core pulls native Whisper/SQLite deps).
  • B. User-action INFO logging — Cockpit.razor: one INFO line per action, each carrying sid + director endpoint: session select (with source= rail / rail-mini / needs-you-nav / deep-link / quick-reply-advance / handover-target / new-session), center & right tab switches, composer Send/Queue/Interrupt/Esc/Attach(name+bytes)/Speak start-stop, kebab actions (incl. hold/wingman), new-session open/submit (repo+options), settings open/save, fan-out.
  • C. Stream + circuit lifecycle — TerminalPane.razor logs connect attempt + dispatched, and a WARNING on empty DirectorBase with the session row state; the child's stdout/stderr is captured by CockpitSupervisor.cs into the Gateway FileLog; and cockpit-terminal.js logs ws connect/open/first-frame(bytes+ms)/close(code,reason)/reconnect, gated behind localStorage.cockpit.debug=1.
