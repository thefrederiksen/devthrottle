Issue #232 - [Build] deploy-cockpit.ps1 misses the scoped-CSS bundle - .razor.css changes deploy stale
Branch loop/232-deploy-cockpit-css · CenCon Development Method · Result: ALL ACCEPTANCE CRITERIA MET

The bug

scripts/deploy-cockpit.ps1 swapped a freshly-published Cockpit into the live per-user install by cherry-picking files: all *.dll/*.pdb, three json configs, wwwroot/app.css, and wwwroot/js/*. It never copied wwwroot/cc-director-cockpit.styles.css - the bundle Blazor compiles every component's scoped *.razor.css into (linked by App.razor) - nor its .br/.gz siblings, nor wwwroot/lib (xterm.css) or wwwroot/pages. So any deploy that changed a *.razor.css shipped with the markup updated but its styling stale (hit for real during the #212 W3 deploy, where the Interrupted-sessions styling lives in Fleet.razor.css - required a manual full-folder swap to recover).

What was implemented

  • Full wwwroot mirror - the issue's preferred option ("cockpit = whole-folder swap"). The cherry-pick of app.css + js/* is replaced by a deterministic mirror of the entire published wwwroot into the live target via robocopy /MIR, wrapped in a new function Sync-CockpitWwwroot. Every static asset - the scoped *.styles.css bundle and its .br/.gz, app.css, js/, lib/, pages/, _framework/ - is always in sync. No copy list to maintain; new assets are picked up automatically.
  • Fail-fast, no fallback (CodingStyle.md) - missing stage throws; robocopy exit codes ≥8 are translated to a thrown error so a real copy failure is loud, not silent. ASCII-only output.
  • The DLL/pdb/json copy logic is unchanged - it already mirrors every app assembly and is correct.
  • A -DefineOnly switch lets the proof test dot-source the script to exercise the real Sync-CockpitWwwroot against temp dirs - one source of truth, no copied logic - without ever touching the live fleet.
  • Hermetic proof test scripts/test/test-deploy-cockpit-css.ps1.

Why a hermetic test instead of a live deploy

The machine is a busy shared host with a LIVE fleet (Gateway 7878, Cockpit 7470, Director 7887) and several other cc-director agent sessions running. Running the real deploy-cockpit.ps1 would shut the live Gateway/Cockpit down. So the proof exercises the exact mirror function the deploy now uses, against throwaway temp dirs that reproduce the real wwwroot shape and the stale-bundle scenario - deterministic, repeatable, and zero impact on the fleet. The test also discriminates: under the OLD cherry-pick logic the scoped bundle stays stale and the test fails; under the fix it passes.
