What was implemented

  • src/CcDirector.Gateway/Voice/GatewayTurnJobStore.cs - in-memory TurnJob store keyed by UUID turn_id; 10-minute TTL checked lazily on read (plus an opportunistic sweep on create); thread-safe (ConcurrentDictionary + per-job lock with snapshot reads). TurnJob fields: TurnId, SessionId, Stage, Transcript, Summary, AudioBase64, ErrorMessage, CreatedAt, ExpiresAt.
  • src/CcDirector.Gateway/Api/GatewayVoiceTurnEndpoint.cs - POST /sessions/{sid}/voice-turn/submit (400 invalid guid, 404 unknown session, 410 exited session, 202 { turn_id, expires_at }) and GET /sessions/{sid}/voice-turn/{turnId} (404 unknown/expired, 200 { stage, transcript, summary, audioBase64, message }). Submit buffers the body (multipart audio/text or raw JSON), resolves the owning Director (SessionOwnerCache fast path, else live fleet fan-out), and fires an unawaited background task that drives the Director's existing SSE POST /sessions/{sid}/voice-turn, mirroring every stage event (submitted/transcribing/transcript/waiting/thinking/summarizing/reply/error) into the job. The task ALWAYS lands the job terminal (reply or error) - a poll can never hang.
  • Wiring: GatewayEndpoints.Map() gained a turnJobs parameter; GatewayHost registers GatewayTurnJobStore as the TurnJobs singleton and passes it through.
  • src/CcDirector.Gateway.Tests/GatewayVoiceTurnAsyncTests.cs - 11 integration tests: real ControlApiHost + real GatewayHost on ephemeral ports, isolated _instancesDir (the WingmanAskForwardingTests pattern), stub QuickIdleBackend/SlowIdleBackend sessions (the VoiceTurnEndpointTests pattern) - no live Claude/OpenAI calls.
