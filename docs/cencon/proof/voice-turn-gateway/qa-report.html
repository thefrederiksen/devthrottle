5. Method / regression checks

  • Thread safety: TurnJob serializes all writes and the snapshot read on one private lock; pollers always consume Snapshot(), never fields piecemeal. The store is a ConcurrentDictionary. No torn reads possible.
  • Background task always terminal: RunTurnAsync is a task entry point owning its try-catch (per CodingStyle: try-catch at entry points only); every exit path lands the job in reply or error - a poll can never hang forever.
  • No fallback programming: failures surface as explicit 4xx/5xx JSON or the job's error stage with the real message; nothing silently degrades.
  • Enterprise logging: FileLog entries on submit, accept, poll-miss, expiry, background start/terminal/failure - all tagged [GatewayVoiceTurn] / [GatewayTurnJobStore].
  • Wiring: GatewayHost.TurnJobs singleton passed through GatewayEndpoints.Map(..., turnJobs:) as an optional parameter (old callers unaffected); specific literal routes win over the #372 catch-all session forwarder - proven live by the passing integration tests, which run with the full route table mapped.
  • Scope respected: no changes to the Director's SSE endpoint, VoiceTurnEndpoint.cs, or ClaudeSummarizer.cs; diff touches only the 6 files declared in the issue.
  • Adjacent surfaces: full Gateway.Tests sweep shows no new failures (Section 3).
