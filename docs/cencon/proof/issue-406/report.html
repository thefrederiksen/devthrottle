Issue #406 - Persist in-flight voice turn-id and resume polling after app restart/background

Developer Agent proof - phone client (.NET MAUI). Branch issue-406-persist-resume-voice-turn.

What was implemented

The in-flight voice turn was tracked only in a local variable inside VoiceConversation.SpeakTurnAsync, so an app kill / background-past-the-foreground-service / crash mid-turn lost the turnId and the poll loop - even though the Gateway still had the completed reply cached for ~10 minutes (GatewayTurnJobStore TTL). This change persists the in-flight turn the instant submit returns and resumes polling it on the next launch, reusing the already-hardened #405 poll loop (no reinvented loop).

  • phone/CcDirectorClient/Voice/IKeyValueStore.cs - new persistence seam (MAUI-free), so the store is unit-testable off-device.
  • phone/CcDirectorClient/Voice/PreferencesKeyValueStore.cs - production impl over MAUI Preferences (app project only).
  • phone/CcDirectorClient/Voice/InFlightVoiceTurn.cs - new record { SessionId, TurnId, SubmittedAt } with JSON round-trip + IsWithinTtl.
  • phone/CcDirectorClient/Voice/InFlightTurnStore.cs - new *Store: single-key persistence (latest wins), ResumeWindow = 10 min aligned to the Gateway job TTL.
  • phone/CcDirectorClient/Voice/VoiceConversation.cs - SpeakTurnAsync gained onTurnSubmitted / onTurnTerminal hooks; new ResumeTurnAsync entry point; shared PollAndSpeakAsync so fresh and resume paths run the identical #405 loop.
  • phone/CcDirectorClient/TalkPage.xaml.cs - persist on submit, clear on terminal, resume-or-discard on OnAppearing; extracted ApplyVoiceTurnUpdate shared by both paths.
  • phone/CcDirectorClient.Tests/InFlightTurnStoreTests.cs - 11 new xUnit tests; csproj link-includes the 3 new MAUI-free files.
