Proof - Issue #590

[Voice] Migrate desktop voice mode to the shared batch pipeline (remove hardcoded whisper-1 + free-text cleanup). Parent #585. Builds on #586 (completeness gate), #587 (shared batch pipeline), #588 (shared component), #589 (desktop dictation batch + the BatchDictationRecorder pattern reused here).

What was implemented

Desktop voice mode is the eyes-free Voice tab (VoiceView) and the full-screen FIFO takeover (FifoWindow). Both previously captured audio through SpeakService, which on the bring-your-own / OpenAI path forced the streaming OpenAiRealtimeProvider (gpt-4o-transcribe, a realtime WebSocket) with live partial transcription, ignoring the user-selected transcription method, and attached a chat-completions cleanup pass on that path.

This change routes both desktop voice surfaces through the SAME whole-audio batch path the Speak dialog uses since issue #589: BatchDictationRecorder. It captures every byte of the turn locally, then on Stop & Send transcribes the whole clip once through the shared BatchTranscriptionPipeline (#587) using the user-selected method (base URL + key + model from the Gateway routing resolver), applies the validated dictionary corrector only (no free-text cleanup), and enforces the completeness gate (#586): an empty/interrupted capture throws NoAudioCapturedException, handled as "nothing heard" rather than transcribing partial input. The Ask Agent versus Ask Wingman choice is a UI-only routing decision applied to the finished transcript and never alters the transcribed text.

Files changed

  • src/CcDirector.Avalonia/Controls/VoiceView.axaml.cs - Voice tab: SpeakService -> BatchDictationRecorder; one batch TranscribeAsync on stop; NoAudioCapturedException handled as "nothing heard"; no live partials.
  • src/CcDirector.Avalonia/FifoWindow.axaml.cs - FIFO takeover: same swap (Ask Agent / Ask Wingman push-to-talk).
  • src/CcDirector.Avalonia.Tests/DesktopVoiceModeBatchProofTests.cs - 5 deterministic proof tests (new).
  • docs/cencon/architecture_manifest.yaml - voice_flow updated to the whole-audio batch path (was OpenAiSttService.TranscribeAsync + free-text summarize).

No new hardcoded endpoint was introduced; the loopback architecture fitness function already allowlists BatchDictationRecorder.cs, and VoiceView/FifoWindow contain no loopback literal, so no guard change was needed. The streaming SpeakService is left intact only for the alpha-hidden wake-word tester surfaces that genuinely need continuous listening; desktop voice mode no longer uses it.
