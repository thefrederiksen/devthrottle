QuestionFinding
Screenshots-file route changed MapGet to Map (any method) - safe? Safe. No other route exists at that template, so nothing is shadowed. Side effect: a POST/PUT to /sessions/{sid}/screenshots/file now forwards to the Director's /screenshots/file with that method instead of the catch-all path - the Director rejects unsupported methods either way, and the surface sits behind the same Gateway auth middleware (mapped after AuthMiddleware when AuthEnabled).
/directors/{id}/settings is Map (any method) - does a stray POST/DELETE do damage? It forwards the method to the Director's /settings, which only maps GET/PUT - other verbs 404/405 on the Director. Equivalent posture to the pre-existing session catch-all. Behind the same auth gate.
Registration order: dirsettings is mapped after the session catch-all - order dependency? None. ASP.NET endpoint routing is precedence-based, not registration-order-based, and /directors/... shares no overlap with /sessions/... templates.
