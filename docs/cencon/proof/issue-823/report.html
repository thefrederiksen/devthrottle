Issue #823 - Developer Agent proof

Title: [Installer] Broken cc-* shims (cc-send/cc-sessions/cc-whoami/cc-ask/cc-spawn/cc-settings/cc-cron/cc-fleet-selftest) point to missing exes; SessionStart banner advertises them

Branch: issue-823-legacy-alias-shims  |  Build: dotnet build cc-director.sln -> Build succeeded, 0 Warning(s), 0 Error(s)

Root cause

The eight retired per-tool fleet commands (cc-send, cc-ask, cc-spawn, cc-sessions, cc-whoami, cc-settings, cc-cron, cc-fleet-selftest) were consolidated into the single cc-devthrottle command, so their per-tool executables no longer ship in pyenv\Scripts. The installer (tools/cc-director-setup-engine/PythonToolsInstaller.cs) writes and removes managed shims only for tools listed in the bundle's tools-manifest.json. The retired aliases are in no current manifest, so the installer never cleaned them up - they survived as orphans in bin\, each pointing at a missing pyenv\Scripts\<name>.exe and failing with exit 127 / No such file or directory.

The SessionStart fleet banner (src/CcDirector.Core/Sessions/FleetPreamble.cs) and the CC_FLEET_TOOLS environment value (src/CcDirector.Core/Sessions/SessionManager.cs) were already migrated to cc-devthrottle exclusively in current code (confirmed below); live sessions seeing the old banner were launched by stale Director builds, an operational matter, not a code defect.

Fix (direction: drop the legacy aliases - the proof-target-sanctioned "absent and replaced in the banner" outcome)

  • PythonToolsInstaller.cs - new LegacyAliasShimNames list + RemoveLegacyAliasShims(), called at the very top of InstallAsync (before the already-installed early-out), so every install and repair purges the orphan .cmd, bare-name bash, and stale .exe shims for the eight retired aliases. The names never overlap the shipping tools, so a live tool's shim is never removed.
  • setup_ops.py - added cc-fleet-selftest to LEGACY_ALIAS_NAMES so the cc-devthrottle setup status doctor diagnostic reports all eight retired names and stays in sync with the installer engine list.
  • docs/architecture/mobile/IMPLEMENTATION.md - corrected the one live doc note that called cc-spawn "broken" to describe it as a retired command (consolidated into cc-devthrottle session spawn).

No banner code change. FleetPreamble.cs and SessionManager.cs already advertise only cc-devthrottle subcommands; the existing FleetPreamble / PiPreamble tests assert the old commands are absent and pass unchanged.
