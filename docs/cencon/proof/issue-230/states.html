Issue #230 - Composer image-upload status states

The four states the patched OnDropImages renders, drawn with the production Cockpit CSS and composer markup. State 1 ("uploading <file> (1 of N)...") is the new optimistic write that now happens before each file's upload await and an immediate StateHasChanged, so a large image on a slow link shows feedback instantly instead of nothing. States 3-4 are the pre-existing completion and failure surfaces (unchanged).
