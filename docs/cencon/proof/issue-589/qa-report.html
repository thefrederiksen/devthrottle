BatchDictationRecorder resolves the method via OpenAiKeyResolver.ResolveEndpointAsync (Gateway vault when attached, local Settings key when standalone). BatchTranscriptionPipeline uses the resolved ResolvedTranscription (BaseUrl/ApiKey/Model) for every request - there is no hard-coded realtime/gpt-4o-transcribe model. Changing the configured method changes BaseUrl/Model, which is exactly what the POST uses. Selected method drives the call. Routing (BaseUrl/ApiKey/Model) comes entirely from the resolver; no fixed model. PASS

No partial; explicit failure on empty/interrupted capture. Empty capture -> loud exception; cancelled turn -> null result. No partial text path exists. PASS
