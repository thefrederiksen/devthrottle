Issue #541 - Whisper.net local transcription as the default

Developer Agent proof report · CenCon Development Method · branch issue-541-local-whisper-default

I BELIEVE THIS IS FINISHED

What was implemented

Local Whisper.net in-process transcription is now the default transcription mode, so the wingman voice screen works offline and out of the box with no API key. OpenAI remains an opt-in bring-your-own-key mode, and DevThrottle is rendered as a disabled "Coming soon" card that can never be selected (enforced both client-side and server-side).

CenCon impact

No architecture or security drift. Local mode is an internal capability that removes the external dependency for the default path (no remote call, no credential) and adds none - the Whisper.net library was already a dependency. The security-critical invariant (the bring-your-own OpenAI key is never paired with the devthrottle.com URL) is preserved by the single pure resolver; local mode carries no key at all, which is strictly safer. No blocking security rule (DT-01..DT-NN) is touched.
