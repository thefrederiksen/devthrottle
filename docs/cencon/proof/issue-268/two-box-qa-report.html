Issue #268 — Two-Box QA Proof

Route per-session WebSocket legs (dictate + terminal stream) through the Gateway so a remote Cockpit never needs a Director's address.

Date: 2026-06-10  ·  Merged commit on main: e388446 (PR #283), atop Cockpit compile fix dd3cd59 (PR #286)
Box A — Gateway host: SOREN_NORTH (tailnet soren-north.taildb08ed.ts.net) — runs the Gateway + Cockpit + the owning Director 37cea6e2 (advertises loopback http://127.0.0.1:7887, the original failing case).
Box B — remote browser/client: SORENLAPTOP (tailnet 100.89.217.71) — a different physical machine, where the per-session WS connections originate.
Deploy: SOREN_NORTH redeployed from main via redeploy-gateway.ps1 + deploy-cockpit.ps1 (Cockpit DLL ts 2026-06-10 16:38). No Director processes killed.

Why this needed two machines

The issue's Proof Target requires every per-session WS proof to be captured "from a non-Gateway machine over the tailnet." Before #268 the browser built the WS URL from the owning Director's advertised address; when that was loopback (127.0.0.1:7887), a remote browser resolved it to its own loopback and the upgrade never landed. The fix builds both WS URLs same-origin to the Gateway, which resolves the owning Director and reverse-proxies the upgrade.
