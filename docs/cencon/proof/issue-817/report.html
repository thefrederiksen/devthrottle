Issue #817 - Mobile Terminal mode: faithful 1:1 translation of the Android terminal tab

A live, headless-safe verification of the React PWA Terminal (mobile/) on a phone-sized viewport (390x844). The app under test is the production Vite build served with the per-machine token injected, exactly as the Gateway serves /m. The backend is a faithful stub that implements the EXACT Gateway contract the app uses (the live /sessions/{sid}/stream WebSocket, /prompt / /escape / /interrupt, and the Gateway-native /wingman/utterance register->chunk->complete transcription flow), so each acceptance criterion is exercised deterministically. Dictation uses Chrome fake audio (--use-file-for-fake-audio-capture) so getUserMedia returns a known clip and the transcript is predictable. No backend change was required: the stream WebSocket is already Gateway-proxied, the browser WebSocket authenticates via the cc-gateway-token cookie the app sets from the injected token, and the utterance flow is the existing Gateway-native equivalent of the Android app's /voice/utterance.

Statement

I believe this is finished. The mobile Terminal mode is a faithful 1:1 translation of the Android terminal tab: a live WebSocket mirror (not polling), collapsible Keys hidden by default, fit-width/1:1 plus A-/A+/pinch zoom, keep-screen-on, the exact Android button set/grouping/payloads, and Speak as dictation (the shared DictationDialog) using the register->chunk->complete utterance flow with NO text-to-speech. AC1-AC5 and AC7-AC12 PASS with committed proof; AC6 is implemented to spec and the wake-lock request is issued and released, but the headful-automation Chrome denies the actual grant (an environment limitation) so it is flagged NOTE for a real-device check by QA.
