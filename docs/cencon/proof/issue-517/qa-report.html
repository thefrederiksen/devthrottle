DEFECT (AC7): Cursor sessions are mislabeled "Claude Code" in the session rail

File (NOT in PR #521 diff - the missing change): src/CcDirector.Avalonia/SessionViewModel.cs
Lines 250-258 and 260-268: 
public string AgentLabel => Session.AgentKind switch
{
    AgentKind.Pi => "Pi",
    AgentKind.Codex => "Codex",
    AgentKind.Gemini => "Gemini",
    AgentKind.OpenCode => "OpenCode",
    AgentKind.RawCli => "Custom CLI",
    _ => "Claude Code"          // <-- AgentKind.Cursor falls through here
};

public ISolidColorBrush AgentBadgeBrush => Session.AgentKind switch
{
    AgentKind.Pi => PiAgentBrush,
    ...
    AgentKind.RawCli => RawCliAgentBrush,
    _ => ClaudeAgentBrush        // <-- AgentKind.Cursor falls through here
};

Steps to reproduce

  1. Configure an enabled Cursor agent.entries row (executable = cursor-agent or a stub on the path).
  2. New Session - select Cursor - pick a repo - Start Session.
  3. Look at the SESSIONS rail (left). Expected: badge reads "Cursor" with a distinct Cursor color. Actual: badge reads "Claude Code" with the Claude-blue brush.

Why this is a fail, not cosmetic

The session rail is the primary place a user identifies which agent a session is running. Every other added provider (Pi/Codex/Gemini/OpenCode) added its AgentLabel/AgentBadgeBrush case when introduced (OpenCode in commit f39315de). This PR added Cursor to the factory, dialog, and drivers but did not update SessionViewModel, so a real Cursor session is indistinguishable from - and actively mislabeled as - a Claude session. The issue's goal is Cursor "selectable... and driven inside a Director session tab exactly like Claude or Pi"; AC7 is not met while the running session misrepresents its agent. The defect is visible in the Developer's own AC7/AC8 screenshots (03/04), where a session running the cursor-agent stub carries a "Claude Code" badge.

Suggested fix (Developer)

Add the missing arms in src/CcDirector.Avalonia/SessionViewModel.cs: AgentKind.Cursor => "Cursor" in AgentLabel, and a dedicated Cursor brush in AgentBadgeBrush (mirroring the Pi/Codex/Gemini/OpenCode pattern). Then re-capture the AC7/AC8 proof showing a "Cursor" badge on the running session.

Evidence
