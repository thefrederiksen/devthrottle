What was implemented

Four recording-state indicators were added to VoiceDictationDialog (the mobile Mode-A voice dialog):

  1. Elapsed timer — m:ss label (Consolas 14pt, muted blue-gray) at top-right. Starts from the moment OnAppearing fires (same frame as StartAsync). Updates on a 100 ms IDispatcherTimer tick.
  2. RECORDING state label — Large label (24pt Bold, red #F44747) centered in the top row. Transitions to "Recorded" (green #5FD08A) immediately when the user taps SUBMIT, before StopAsync completes, giving instant visual confirmation.
  3. Pulsing dot — 16px red circle (BoxView, CornerRadius=8) at top-left. Toggles Opacity between 1 and 0 on a 500 ms IDispatcherTimer tick (~1 Hz). Hidden (IsVisible=false) on SUBMIT/CANCEL/back-gesture. Stopped in OnDisappearing and ResolveAsync.
  4. Speak-up hint — Amber label (#D97706, 13pt) below the bouncing bars. Fires when ReadLevel() returns below VoicePresentLevel=0.025 (mirrors desktop threshold: sqrt(20/32767) ~ 0.025 on the 0..1 sqrt-scaled range) for more than 2 continuous seconds. Hides immediately when level recovers.

All timers share the same StopTimers() helper called from both OnDisappearing and ResolveAsync to ensure no timer leaks.
