AC1 - Status label order: Transcribing -> Thinking -> Summarizing -> Speaking

Defect that was filed

In VoiceConversation.SpeakTurnAsync, the "reply" TurnUpdate was setting VoiceStatusLabel.Text = "Speaking..." in TalkPage's reply handler, then "summarizing" fired after it - producing the inverted sequence Thinking... -> Speaking... -> Summarizing... instead of the required Thinking... -> Summarizing... -> Speaking...

Fix applied (commit 9362343)

Two-part fix:

  1. VoiceConversation.cs: Added an explicit "speaking" TurnUpdate emitted AFTER SummarizeForVoiceAsync completes and BEFORE SpeakAsync is called. The comment on the "reply" event was updated to document that it carries raw text for the screen label only and does NOT advance the status indicator.
  2. TalkPage.xaml.cs: Removed the premature VoiceStatusLabel.Text = "Speaking..." from the "reply" handler. Added a dedicated "speaking" case that sets the status to "Speaking..." when TTS actually begins.

Code evidence - VoiceConversation.cs (lines 107-125)

var rawReply = result.SpokenText();

// "reply" updates the on-screen reply label only - it does NOT advance the
// status indicator. The status must follow the actual processing order:
// Thinking -> Summarizing -> Speaking. See AC1.
onUpdate?.Invoke(new TurnUpdate("reply", rawReply));

// Step 5: summarize the raw reply into 2-3 spoken sentences of plain prose.
onUpdate?.Invoke(new TurnUpdate("summarizing", "Summarizing..."));
var spokenSummary = await SummarizeForVoiceAsync(session, rawReply, ct);

// "speaking" fires after summarization completes and before TTS begins,
// so the status label reads "Speaking..." only while audio is actually playing.
onUpdate?.Invoke(new TurnUpdate("speaking", "Speaking..."));
await SpeakAsync(session.TailnetEndpoint, spokenSummary, ct);

Code evidence - TalkPage.xaml.cs (lines 484-491)

case "reply":
    // Update the on-screen reply label only.
    // Status is updated by the "speaking" stage after summarization.
    VoiceReplyLabel.Text = u.Text;
    VoiceReplyCard.IsVisible = true;
    break;
case "speaking":
    VoiceStatusLabel.Text = "Speaking...";
    break;
