Issue #722 - cc-fleet-selftest - Developer Proof

An on-demand end-to-end smoke test of fleet session messaging: spawn two throwaway sessions, list, send, ask, tear down, PASS/FAIL.

What was implemented

  • tools/cc-fleet-selftest/ - spawns a deterministic "responder" (cmd with a custom prompt marker FLEETPONG) + a recipient via POST /sessions, then exercises GET /fleet/sessions, POST /fleet/send, POST /fleet/ask (the machinery the cc-* tools wrap), asserts each, tears the sessions down (DELETE /sessions/{id}), prints PASS/FAIL, exits 0/non-zero.
  • cc_shared/director.py - added delete().
  • Shipped + verified: tools/registry.json (ship:true), Core tools-manifest.json, docs/cli-reference.md, and the FleetToolsShipGuardTests guard array.
