1. What Was Implemented

Added AgentKind.RawCli = 5 and a new RawCliAgent class that lets Director spawn any user-supplied CLI (aider, pwsh, cmd, etc.) in a ConPty terminal session. The custody layer (ConPty, buffers, keystrokes, git, cc-* tools) is agent-agnostic; only Claude-specific interpretation (turn-detection heuristics, wingman briefing, transcript linking) needed to degrade gracefully - which it does because those features key off Claude-specific output patterns that a raw shell never produces.

  • AgentKind.cs: RawCli = 5 added with XML doc explaining raw-terminal semantics
  • RawCliAgent.cs: IAgent implementation; SupportsPreassignedSessionId = false, SupportsStudioMode = false; resume/studio flags silently ignored with log entries; construction-time extra args combine cleanly with user args
  • NewSessionRequest.cs: Command and CommandArgs fields added (flagged assumption per issue: exact contract shape is the developer's design)
  • SessionDto.cs: Agent field comment updated to include RawCli
  • ControlEndpoints.cs POST /sessions: validates command is required for RawCli, builds RawCliAgent(req.Command, req.CommandArgs); 400 with descriptive error when missing; error message for valid agent list updated
  • NewSessionDialog.axaml: "Custom CLI" radio button added to agent picker; CustomCliPanel with Command + Args input boxes shown only when Custom CLI selected; always visible (not alpha-gated - user explicitly provides the exe)
  • NewSessionDialog.axaml.cs: SelectedCustomCommand, SelectedCustomArgs properties; AgentRadio_CheckedChanged shows/hides CustomCliPanel; UpdateActionButton requires non-empty Command for Start to enable; BtnAction_Click captures custom fields
  • MainWindow.axaml.cs: New CreateSession(IAgent agent, ...) overload; ShowNewSessionDialog builds RawCliAgent for RawCli kind; preflight uses RawCli-specific "command not found" error message
  • SessionViewModel.cs: AgentLabel = "Custom CLI", AgentBadgeBrush = slate gray for RawCli
  • RawCliAgentTests.cs: 18 unit tests (constructor validation, Kind, capabilities, ExecutablePath, BuildLaunchSpec)

5. CenCon Impact

Architecture: No architecture drift. RawCliAgent is a new leaf node in the existing IAgent strategy pattern. No new containers, no new cross-boundary calls, no new ports or protocols. The existing custody layer (ConPty, SessionManager, ControlEndpoints) is reused without modification to its core contract.

Security: No blocking security rule violations. The exe path is logged at launch (audit trail). No allowlist on what exe may be run - this is the user's machine and the endpoint is already token-gated; the issue explicitly flags this assumption. The exe is resolved via ExecutableResolver before spawning (fails loudly if not found), consistent with all other agents.

CONTRACT_AUDIT.md: No new routes added. POST /sessions route count unchanged at 115. Only the agent validation and construction logic inside the existing route was changed.
