Regression / method sweep

  • Slim-by-default poll is back-compatible: the field audioBase64 is always present (shape unchanged), null by default, populated only for ?includeAudio=1 / X-Include-Audio - an old phone keeps working for one release. OK
  • No fallback programming: a base64 that fails to decode stores zero bytes and the endpoint returns an honest 404 (logged), not invented audio. OK
  • Token gate enforced at the endpoint even with authEnabled=false (production tray mode) - missing token = 401 on both poll and audio routes. OK
  • No security rule (DT-*) implicated; the change is an in-memory transport refactor of already-authorized data. OK
