Issue #859 - Stop turn-end re-narration when voice is turned off

Developer Agent proof. Branch issue-859-gateway-voice-unmark (off main which already has #850).

I believe this is finished.

1. What was implemented

  • Gateway service (src/CcDirector.Gateway/Wingman/WingmanVoiceService.cs): new Unmark(sid) - the counterpart to Mark(sid). It removes the session from the persisted voice-session set, drops the cached spoken summary + audio (and the durable copy on disk), and clears the transient generating marker. Read-only: it sends nothing into the session.
  • Gateway endpoint (src/CcDirector.Gateway/Api/GatewayWingmanVoiceEndpoint.cs): POST /sessions/{sid}/wingman/voice/stop -> { stopped: true }. Validates the session id, calls Unmark; idempotent (stopping a non-voice session is a 200 no-op).
  • Mobile wiring (mobile/src/api/client.ts, mobile/src/pages/VoiceMode.tsx): a new stopWingmanVoice() call, and "Turn voice off" now calls it alongside the existing Director POST /voice-mode { enabled:false }.

Why this stops the spend without touching the watcher: the turn-end watcher (GatewayHost turn-end handler) and the background voice sweep (SweepVoiceSessionsAsync) already gate on IsVoiceSession / VoiceSessionIds. Removing the session from the set makes both skip it - no code change needed there.
