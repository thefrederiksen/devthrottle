Issue #838 - Mobile roster cards: full-width wrapping name (no truncation), repo moved below the name

Developer Agent QA proof - CenCon Development Method. Surface: mobile PWA (/m), files mobile/src/pages/Home.tsx (SessionRow) and mobile/src/styles.css. Mobile-only layout change, owner-approved "Option A". No backend change.

What was implemented

  • Name now wraps, full width, no truncation. Removed the white-space:nowrap; overflow:hidden; text-overflow:ellipsis clamp on .row-name; added overflow-wrap: anywhere and an explicit line-height. The name now spends the full card width and wraps to as many lines as it needs.
  • Repo moved out of the right of the name row to below the name. The SessionRow markup wraps the status text and the repo in a new .row-meta line under the name: <status> | <repo>. The repo is no longer a right-pinned pill - it is secondary text (smaller, dimmer via opacity) but fully readable. A thin CSS-drawn vertical divider (.row-divider, 1px, no glyph) separates the two.
  • Status dot top-aligned at the start of the first name line. .row-link changed from align-items:center to align-items:flex-start, with a small margin-top on .dot so it centers on the first line's text even when the name wraps to several lines.
  • Everything else preserved: the "Needs you" group above "All sessions", the needs-you attention border, the status dot colors, and tap-anywhere-to-open (the whole card is still the <a class="row-link"> link). No change to the data, ordering, or the GET /sessions typed client.

How it was verified

The built mobile app (tsc --noEmit + vite build, clean) was served via vite preview and driven by a headless browser at a 390 x 844 phone viewport. A crafted roster (short needs-you, medium needs-you, a long-named working session, a short working session) was returned for GET /sessions; the per-machine Gateway token was injected into index.html exactly as the Gateway does at serve time, so the typed client attached its Authorization: Bearer header. The "before" column is the unchanged main version of the same two files, built and screenshot identically.
