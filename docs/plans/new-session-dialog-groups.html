Session types — badges & behavior

Badge
Type
Behavior
·
Developer
Write code, fix bugs, ship. Default; no badge on the rail. (was "Implement")
P
Product
Scope the work, file issues for the developer to implement. Never codes. (was "Bug Report")
Q
QA
Verify against what was asked; report findings, never fix.
S
Support NEW
Triage incoming questions/support, answer what it can, file issues for real bugs. Never edits code.
D
Discuss
Talk only — never edits or commits.

Five types total. The Product group uses four of them — Product, Developer, QA, Support; Discuss is single-session only. "Implement" is renamed Developer and "Bug Report" is renamed Product; "Issue Submitter" is dropped, which frees the S badge for Support.
