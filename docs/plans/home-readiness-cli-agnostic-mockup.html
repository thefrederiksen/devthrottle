CC DIRECTOR — Home Readiness Card

The readiness card no longer hard-requires Claude Code. It now reports whichever agent CLIs are installed (Claude Code, Pi, Codex, Gemini, OpenCode) and only goes red when none are present. The OpenAI key and Director rows are removed.

Before

4 rows • Claude hard-required • key miscolors card red
READINESS 3 of 4 ready
OK claude CLI 2.1.177 (Claude Code) - on PATH
OK OpenAI key Set (Gateway vault)
X cc-* tools 0 of 32 on PATH
OK Director v0.6.23 (7f64108) - running
Problems:
• claude CLI assumes Claude specifically — a Codex/Pi-only box reads as broken.
• OpenAI key is voice-only, yet a missing key paints the whole card red / "not ready".
• Director is always OK — pure filler; version is already in the footer.

After

2 rows • any CLI counts • red only if none installed
SYSTEM STATUS ALL GREEN
OK Agent CLIs Claude Code 2.1.177, Codex - on PATH
! cc-* tools 0 of 32 on PATH
Agent CLIs = green when one or more of the five are installed (lists them). cc-* tools is a soft capability line (warn, never blocks). With at least one CLI present and tools ignorable, the card reads ALL GREEN.

After — the only red case

no agent CLI at all
READINESS 1 of 2 ready
X Agent CLIs No agent CLI found - install Claude Code, Codex, Pi, Gemini, or OpenCode
! cc-* tools 0 of 32 on PATH
The Agent CLIs row only goes red when zero of the five are detected. "Fix it" opens Settings > Tools.

After — Codex-only box

no Claude, still ready
SYSTEM STATUS ALL GREEN
OK Agent CLIs Codex 0.21.0 - on PATH
OK cc-* tools 32 of 32 on PATH
No Claude Code installed, but Codex is — the box is fully ready. This is the case that was impossible before.

Gateway card (the green "GATEWAY CONNECTED" panel above the readiness card) is unchanged — still its own card, still informational, not a counted requirement.
