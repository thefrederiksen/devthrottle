Behavior summary

  • Chrome always on. The left rail (New Session, gateway light, session list) is present whether or not a session exists. No more full-window takeover.
  • Status = the main area when no session is selected. Like VS Code's Welcome tab — it's not a mode, it's just what fills the pane when nothing else is focused.
  • Quiet when healthy, loud when not. All-green = one line. Any failure = only the failing checks, with fixes.
  • Recent / history removed from the local Director — that moves to the Gateway. Local keeps only must-haves.
  • Gateway indicator stays at the bottom of the rail (unchanged from today) — glanceable health at all times. The full Status screen is summonable any time (View ▸ Status, or deselect the session) without closing your session.
