CC Director - Screen Feature Review

Simplification pass · 2026-06-15 · build v0.6.23

Architecture principle. CC Director is a dumb, thin wrapper around the Claude Code CLI - the always-available "go through the metal" escape hatch. It must stay simple and never break, so that whatever else fails, the user can always drive a session directly.

The Cockpit is where we want to spend our time. Everything that is supervisory, fleet-wide, cross-machine, or "rich dashboard" in nature belongs up a level in the Cockpit. The Director keeps only: the raw terminal, send / speak / queue, minimal local session management, and the CLI-configuration plumbing.

Recommended next steps (in priority order)

  1. Delete the dead/legacy. Remove the Agent (legacy) tab + empty panel, the "Director (multi-session)" Tools item + hidden right-panel tab, the hidden header Refresh button, and the duplicate Session>New Session and Tools>Settings menu items. Pure subtraction, zero user-visible regression.
  2. Formalize "moved up" features. For each Move up item, confirm the Cockpit covers it, then drop it from the Director menus entirely (rather than leaving it hidden-but-wired).
  3. Slim the menus. Collapse the four File > "Open X" items, and audit the 10-item session kebab.
  4. Make Cockpit the obvious destination. The Cockpit toolbar button is the bridge - give it visual weight so users naturally graduate to it.
