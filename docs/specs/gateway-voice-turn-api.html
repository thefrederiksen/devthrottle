Gateway Voice Turn API

Async voice-to-agent round-trip over a resilient, chunked-upload pipeline

spec v1.0  |  2026-06-11  |  cc-director

Overview

A voice turn lets a client (phone, web app, car) speak to a running agent session and hear a spoken reply. The pipeline must survive poor network conditions: audio is uploaded in resumable chunks and all processing happens on the Gateway after the upload is complete. The client just submits audio and waits; a status poll tells it when audio is ready to play.

Design principle: The client does one thing -- capture audio and play audio. Everything else (transcription, routing, agent response, TTS synthesis) happens on the Gateway, making the same REST surface reusable by any client: phone, browser, car unit, CLI test script.

Client (any)

  • Records audio (or types text)
  • Uploads in chunks -- survives signal drops
  • Polls for status
  • Downloads and plays response audio

Gateway (port 7878)

  • Accepts chunked audio upload
  • Transcribes with Whisper
  • Routes text to the right Director / session
  • Synthesizes agent reply to audio with TTS
  • Stores result until client downloads

End-to-End Flow

Turn State Machine

State Meaning Transition
registered Turn ID allocated. No chunks yet. First chunk received → uploading
uploading Chunks are being accepted. POST /complete with all chunks present → transcribing
transcribing Audio assembled, Whisper running. Transcript received → thinking
thinking Text forwarded to agent session, waiting for reply. Agent turn complete → synthesizing
synthesizing Reply being summarized and converted to speech. Audio ready → ready
ready Audio available for download. Terminal state. Client downloads, then client calls DELETE (optional cleanup)
error Pipeline failed at any stage. Terminal state. errorMessage and errorStage fields are set

REST API Reference

All endpoints are on the Gateway. Default local address: http://localhost:7878. Remote (Tailscale): https://soren-north.taildb08ed.ts.net:7878.

All endpoints require the Gateway auth token header: X-Director-Token: {token} (same token used by Directors to register).

POST /voice-turn

Register a new voice turn. Returns a turnId the client uses for all subsequent calls. Optionally accepts text directly (skips upload and transcription phases).

Request Body (JSON)

{
  "sessionId":  "d3f1a2b4-...",     // required -- target agent session
  "directorId": "ce7e7143-...",     // optional -- if omitted, Gateway locates it
  "text":       "what are you working on?" // optional -- skips audio upload entirely
}

Response 201 Created

{
  "turnId": "a1b2c3d4e5f6...",
  "state":  "registered"          // or "thinking" if text shortcut used
}

Errors

400Missing sessionId
404Session not found on any connected Director
PUT /voice-turn/{turnId}/chunk/{index}

Upload one audio chunk. Idempotent: uploading the same chunk twice (same SHA256) is a no-op. Safe to retry on signal drop. Chunks are zero-indexed.

Request

HeaderValue
Content-Typeapplication/octet-stream
X-Chunk-Sha256Optional -- hex SHA256 of the chunk bytes (integrity guard)

Body: raw audio bytes (any supported format: webm, mp3, wav, m4a, ogg, aac, flac)

Response

204Chunk accepted
409SHA256 mismatch -- client should re-upload
422Turn is in a state that no longer accepts chunks (already complete/error)
POST /voice-turn/{turnId}/complete

Tell the Gateway that all chunks are uploaded. The Gateway validates all chunks are present, then starts the pipeline (assemble → transcribe → think → TTS). Returns immediately with 202; the client polls /status to follow progress.

Request Body (JSON)

{
  "totalChunks": 6,
  "mime":        "audio/webm"
}

Response

202Pipeline started. Poll /status.
409Missing chunks -- body lists which ones: { "missingChunks": [2, 5] }
GET /voice-turn/{turnId}/status

Poll pipeline progress. Recommended interval: every 1-2 seconds. When state is ready or error polling can stop.

Response 200

{
  "turnId":      "a1b2c3d4...",
  "state":       "thinking",         // registered | uploading | transcribing | thinking | synthesizing | ready | error
  "transcript":  "what are you working on?", // set when transcription completes
  "summary":     null,                // set when synthesis completes (the spoken text)
  "errorMessage": null,               // set on error state
  "errorStage":   null,               // which stage failed: "transcription" | "routing" | "agent" | "tts"
  "createdAt":   "2026-06-11T20:30:00Z",
  "updatedAt":   "2026-06-11T20:30:14Z"
}
GET /voice-turn/{turnId}/audio

Download the synthesized response audio. Only available when state is ready.

Response

200Content-Type: audio/mpeg -- MP3 bytes, ready to play
425Turn not yet ready (still processing)
410Turn in error state -- audio will never be available
DELETE /voice-turn/{turnId}

Clean up a completed or errored turn. Removes stored chunks and audio from disk. Optional -- the Gateway auto-purges turns older than 24 hours.

204Deleted
422Turn is still in progress (cannot delete active turns)
GET /voice-turns

List all recent voice turns (last 24 hours). Useful for monitoring and debugging.

Response 200

[
  {
    "turnId":    "a1b2c3d4...",
    "sessionId": "d3f1a2b4-...",
    "state":     "ready",
    "createdAt": "2026-06-11T20:30:00Z"
  }
]

Design Notes

Bad reception resilience

Chunks are stored the moment they land. If the phone drops signal mid-upload, it simply re-sends the missing chunks later. The SHA256 guard means re-sending a chunk that already arrived is a free no-op. Only when POST /complete arrives with a valid chunk count does the pipeline start.

Text shortcut (test + web)

Including "text" in the POST /voice-turn body skips audio upload and transcription entirely. The turn goes straight to thinking. This is the primary path for the test harness and for web app text-input mode.

Gateway as hub

The Gateway routes to the right Director based on sessionId. If directorId is omitted, the Gateway queries its registered Directors to locate the session. This means the client never needs to know which Director is running the session.

Notification (future)

The current design uses polling. A future enhancement can add a webhook or SSE event when the turn becomes ready, so the client does not need to poll. The phone can show a "response waiting" indicator without keeping a connection open.

What does NOT live on the phone / client: transcription, agent routing, summarization, TTS. The client only records audio bytes (or types text), uploads them, and plays back the response MP3. All intelligence is on the Gateway.

Test Harness

These scripts let you drive the full voice-turn pipeline from a terminal -- no phone required. The text shortcut path (step 1b) is the fastest way to test that routing, agent communication, and TTS are all working.

Prerequisites: Gateway running on localhost:7878, at least one Director connected with a live session, $TOKEN set to the Gateway auth token, $SESSION set to a session GUID.

Path A: Text shortcut (no audio)

Fastest test. Skips recording and Whisper entirely.

# PowerShell

$GW = "http://localhost:7878"
$TOKEN = "your-gateway-token"   # from gateway config.json
$SESSION = "d3f1a2b4-..."        # session GUID from Director /sessions
$HEADERS = @{ "X-Director-Token" = $TOKEN; "Content-Type" = "application/json" }

# 1. Register turn with text
$body = @{ sessionId = $SESSION; text = "What are you currently working on? One sentence." } | ConvertTo-Json
$turn = Invoke-RestMethod -Uri "$GW/voice-turn" -Method POST -Headers $HEADERS -Body $body
$tid = $turn.turnId
Write-Host "Turn registered: $tid, state: $($turn.state)"

# 2. Poll until ready
do {
    Start-Sleep -Seconds 2
    $status = Invoke-RestMethod -Uri "$GW/voice-turn/$tid/status" -Method GET -Headers $HEADERS
    Write-Host "  state: $($status.state)"
} while ($status.state -notin "ready", "error")

if ($status.state -eq "error") {
    Write-Host "ERROR at $($status.errorStage): $($status.errorMessage)"
    exit 1
}

Write-Host "Spoken reply: $($status.summary)"

# 3. Download audio
$dl = @{ "X-Director-Token" = $TOKEN }
Invoke-WebRequest -Uri "$GW/voice-turn/$tid/audio" -Headers $dl -OutFile "reply.mp3"
Write-Host "Audio saved to reply.mp3"
Start-Process "reply.mp3"   # opens in Windows Media Player

# 4. Cleanup
Invoke-RestMethod -Uri "$GW/voice-turn/$tid" -Method DELETE -Headers $HEADERS

Path B: Chunked audio file (full pipeline)

Splits a local audio file into chunks and drives the complete pipeline including Whisper.

# PowerShell -- supply any audio file

$GW = "http://localhost:7878"
$TOKEN = "your-gateway-token"
$SESSION = "d3f1a2b4-..."
$AUDIO_FILE = "D:\test\question.m4a"  # any format Whisper accepts
$CHUNK_SIZE = 128KB                   # 128 KB per chunk

$HEADERS = @{ "X-Director-Token" = $TOKEN }

# 1. Register
$h = $HEADERS + @{ "Content-Type" = "application/json" }
$turn = Invoke-RestMethod -Uri "$GW/voice-turn" -Method POST -Headers $h `
          -Body (@{ sessionId = $SESSION } | ConvertTo-Json)
$tid = $turn.turnId

# 2. Split and upload chunks
$bytes = [System.IO.File]::ReadAllBytes($AUDIO_FILE)
$total = [Math]::Ceiling($bytes.Length / $CHUNK_SIZE)
for ($i = 0; $i -lt $total; $i++) {
    $start = $i * $CHUNK_SIZE
    $len = [Math]::Min($CHUNK_SIZE, $bytes.Length - $start)
    $chunk = $bytes[$start..($start + $len - 1)]
    $sha = [BitConverter]::ToString(
        [System.Security.Cryptography.SHA256]::Create().ComputeHash($chunk)
    ).Replace("-", "").ToLower()
    Invoke-RestMethod -Uri "$GW/voice-turn/$tid/chunk/$i" -Method PUT `
        -Headers ($HEADERS + @{ "X-Chunk-Sha256" = $sha; "Content-Type" = "application/octet-stream" }) `
        -Body $chunk
    Write-Host "  chunk $i/$($total-1) uploaded"
}

# 3. Complete
$ext = [System.IO.Path]::GetExtension($AUDIO_FILE).TrimStart(".")
$mime = @{ m4a="audio/mp4"; mp3="audio/mpeg"; wav="audio/wav"; webm="audio/webm"; ogg="audio/ogg" }[$ext]
Invoke-RestMethod -Uri "$GW/voice-turn/$tid/complete" -Method POST `
    -Headers ($HEADERS + @{ "Content-Type" = "application/json" }) `
    -Body (@{ totalChunks = $total; mime = $mime } | ConvertTo-Json)

# 4. Poll + download (same as Path A above)

curl (cross-platform)

# Register a text-mode turn
curl -s -X POST http://localhost:7878/voice-turn \
  -H "X-Director-Token: {token}" \
  -H "Content-Type: application/json" \
  -d '{"sessionId":"d3f1a2b4-...","text":"What are you working on?"}' | jq .

# Poll status
curl -s http://localhost:7878/voice-turn/{turnId}/status \
  -H "X-Director-Token: {token}" | jq .

# Download audio
curl -s http://localhost:7878/voice-turn/{turnId}/audio \
  -H "X-Director-Token: {token}" -o reply.mp3
