FIXED: #106 - cc-powerpoint drops numbered-list body

Status: Fixed, rebuilt, deployed, and verified.

The bug

In cc-powerpoint from-markdown, a slide using a numbered list (1. 2. 3.) produced a slide with only the title - the entire body was missing. Bullet lists (- ) worked fine.

Root cause

tools/cc-powerpoint/src/parser.py -> _extract_bullets() only tracked bullet_list_open / bullet_list_close tokens from the markdown-it token stream. Numbered lists emit ordered_list_open / ordered_list_close, which were ignored, so list_depth never rose and no body items were collected.

The fix

if token.type in ("bullet_list_open", "ordered_list_open"):
            list_depth += 1
        elif token.type in ("bullet_list_close", "ordered_list_close"):
            list_depth -= 1

Added two regression tests (test_numbered_list, test_numbered_sub_bullets) to tests/test_parser.py. All 19 parser tests pass.
