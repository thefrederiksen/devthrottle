What this is

A standalone Android app records audio fully offline into crash-safe rolling segments, then uploads them to the always-on CC Director Gateway, which transcribes each segment through the existing dictation pipeline, cleans it with your company-term glossary, and files the result into a dedicated vault transcripts collection. The phone stays a dumb, reliable recorder; all the heavy lifting is server-side.

The app, running on an emulator

Built with .NET MAUI, deployed to a freshly-provisioned Android 35 emulator, and driven through a real record cycle.

The emulator microphone records silence (no host audio source), but the capture, segment finalization, note timestamping, and manifest writing are all real — see the manifest below.

What YOU need to test (hardware / physical)

Everything that can be verified headless has been. These three need your real phone — they cannot be done in an emulator or unattended.

  1. Real-microphone, multi-hour, crash-safety run. Install the APK on your phone, start a long recording (target the real 2-3 hour case), lock the screen and background the app, then force-kill it mid-segment. Confirm every prior 1-minute segment is on disk and playable, and that no audio is lost. This is the core promise; the emulator can't prove it.
  2. End-to-end over your tailnet. Put the phone on your tailnet, enter your Gateway HTTPS URL + token in the app, record a short real clip, hit "Upload latest recording", and confirm the transcript appears in the vault (cc-vault docs list) with the audio attached.
  3. APK install + permissions. Sideload the signed APK (enable "install unknown apps"), grant the microphone permission on first Record, and confirm the persistent "Recording" notification appears.

How to build & run it yourself

The Android app

# Build a self-contained debug APK (assemblies embedded, installable standalone)
dotnet build phone/CcRecorder/CcRecorder.csproj -f net10.0-android -p:EmbedAssembliesIntoApk=true

# Install on a connected device/emulator
adb install -r phone/CcRecorder/bin/Debug/net10.0-android/com.ccdirector.recorder-Signed.apk

APK id com.ccdirector.recorder. For distribution, build -c Release with a signing keystore.

The server (ingest endpoints live in the Gateway)

The /ingest/recording routes are mapped into the Gateway automatically. To run the real end-to-end test that files into the vault:

setx OPENAI_API_KEY ...        # already set on this machine
$env:CC_REC_E2E = "1"
dotnet test src/CcDirector.Gateway.Tests --filter RecordingEndpointsE2ETests

Notes & honest gaps

  • Toolchain installed during this run: the .NET MAUI Android workload, a user-local Android SDK (%LOCALAPPDATA%\Android\Sdk), platform-tools, the emulator, and an Android 35 system image. ANDROID_HOME now points there.
  • Progressive transcription (transcribing early segments while the call continues) is designed for but not yet wired — the server currently transcribes all segments at complete. The chunked design makes this a drop-in later.
  • Foreground-service ownership: the recorder runs capture in-process with a foreground service keeping it alive. For the strongest multi-hour guarantee, moving the MediaRecorder loop fully inside the service is a hardening follow-up — flagged for the crash-safety test above.
  • Not committed. All changes are left uncommitted for your review, per house rules.
