2. Issue #244 - Terminal view must work 100%

The problem

The Terminal tab wrapped an xterm.js WebView inside an outer MAUI ScrollView, with the WebView pinned to a fixed HeightRequest="600". Two defects fell out of that:
  • Nested scroll conflict. The WebView is itself a scroll container, so the outer page fought its drag gesture - "really hard to move back and forth."
  • Fixed 600px height. The terminal never filled the screen; the live input box was often off-screen - "it doesn't show the whole screen."
  • At the PTY's true column width the grid is far wider than a portrait phone, so the whole width was never visible at once either.

The fix

  • Removed the outer ScrollView and the fixed height. The terminal is now in a Grid Auto,*,Auto and fills all remaining vertical space; the WebView is the only scroll container and owns both axes.
  • Added a Fit width mode (default on): the xterm font shrinks so all PTY columns map onto the WebView width - the whole width shows with no horizontal pan. One tap on the 1:1 toggle returns to full resolution (horizontal pan via the page). Font sizing is a real layout change, so scroll geometry stays correct (unlike a visual CSS scale).
  • The control keys (input, Send, Enter/Esc/Stop, arrows) collapse behind a Keys toggle so the terminal is maximized by default.
  • The existing safeguard - never shrink rows below the PTY's row count, so Claude Code's cursor-up redraws do not stack ghost copies - is retained and unit-tested.

3. Issue #245 - Start a new session from recent repos

What was built

  • A + New session button on the roster opens a two-step picker: 1. Machine (any Director in the fleet, from GET /directors) then 2. Repository (that Director's recent repos, newest-first, from GET /directors/{id}/repos), plus an "Or enter a path" fallback.
  • The most-recently-seen machine is auto-selected and its repos load immediately. Switching machine reloads that machine's repos.
  • Tapping a repo (or Create with a typed path) calls POST /directors/{id}/sessions, shows a Creating... busy state, and on success opens the new session straight on the Terminal tab.
  • New pure-logic classes DirectorInfo, RepoInfo, FleetParser and 3 GatewayClient methods. No backend changes - the Gateway already exposed every endpoint.

5. Test environment, scope, and cleanup

Environment. Android emulator emulator-5554; app pointed at a live Gateway via adb reverse tcp:7878 (gateway) and tcp:7889 (local Director). The gateway reported 3 real Directors (SOREN, SOREN_NORTH, SORENLAPTOP).

Scope notes (honest).

  • The terminal stream and the end-to-end create were exercised against the local development Director (SOREN_NORTH, 127.0.0.1:7889), the only Director reachable from the emulator without Tailscale. Remote Directors (SOREN, SORENLAPTOP) are tailnet HTTPS endpoints; on a real phone the device is on the tailnet and reaches them directly - that path is the same code, not separately re-run here.
  • The create test used a throwaway folder D:\ReposFred\_qa_newsession_test. Cleanup done: the session was killed + removed (DELETE /sessions/{sid} -> {"killed":true,"removed":true}), the folder de-registered from the Director's recent list, and the folder deleted - all verified gone.
