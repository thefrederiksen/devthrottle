Review Rules

Shipped Tool Standard

  • Must appear in the install bundle and tools manifest intentionally.
  • Must have a reliable --version or equivalent health check.
  • Must have a zero-auth smoke test, or a documented reason why that is impossible.
  • Must have current docs matching the real command surface.

Extra Tool Standard

  • Stays buildable in the repo but is not promised to installed users.
  • Needs a clear owner, purpose, dependency list, and test/proof before shipping.
  • Tools that require personal API keys or heavyweight local dependencies should stay extra unless there is a product reason to ship them.
  • Promotion means updating registry ship status, manifest, docs, tests, and release bundle proof together.
