DevThrottle Tool Inventory Review

Review artifact for the current cc-* command surface. It separates the product tools shipped to users from extra repo tools, then calls out directories under tools/ that are not part of the registry and need an explicit keep, merge, retire, or register decision.

Sources: tools/registry.json, src/CcDirector.Core/Tools/tools-manifest.json, and the tools/ directory. Last refreshed: 2026-06-27 after folding cc-settings into cc-devthrottle settings.

Current Read

What Ships

  • The product surface is the registry allowlist where ship: true.
  • The shipped manifest has the same 9 tools and owns catalog health checks.
  • cc-devthrottle is the single shipped Fleet tool for sessions, messages, settings, schedules, and setup operations.
  • cc-settings is no longer a standalone CLI. The separate cc-settings-api Claude skill remains because it drives live Control API writes.

What Needs Review

  • Every extra registered tool needs a keep, merge, retire, or promote decision.
  • Promotion requires registry ship status, shipped manifest entry, docs, installer inclusion, and passing proof.
  • Unregistered command-like directories are the biggest cleanup gap because they are neither explicitly extra nor intentionally shipped.
  • Auth-heavy and platform-specific tools should stay extra unless the product has a clear reason to support them.
