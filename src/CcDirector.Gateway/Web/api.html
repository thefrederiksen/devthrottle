CC DIRECTOR

Gateway API reference directory healthz

Overview

The Gateway is the network-wide hub for the CC Director fleet. It aggregates read-only state from every registered Director so a single browser tab or programmatic consumer can see what is running across the network of agent sessions.

Writes go direct to the Director. To send a prompt, rename a session, interrupt, or delete, call the owning Director directly using the tailnetEndpoint and sessionId fields returned by GET /sessions. The Gateway does not proxy writes.

Authentication

All endpoints below (except /healthz) require either a session cookie set via POST /login or an Authorization: Bearer <token> header. The token lives in gateway-token.txt next to the Gateway executable.

Endpoints

GET/sessions

Returns every session across every registered Director. Each session carries the owning machine's name and a precomputed viewUrl deep-link so consumers can talk to it directly.

Query parameters

NameTypeDefaultDescription
envelopeboolfalseWhen true, wrap the result in { sessions, machineErrors } so unreachable Directors are visible. Otherwise return a flat array (backward compatible).
includeExitedboolfalseInclude sessions in the Exited activity state. Hidden by default to match the directory UI.
statusColorstring-Filter by aggregate status: red | yellow | green | unknown.
agentstring-Filter by agent kind: ClaudeCode | Pi | Codex | Gemini.
statestring-Filter by exact activityState.
machinestring-Filter by Director machine name (exact, case-insensitive).
directorstring-Filter by Director GUID (exact, case-insensitive).
qstring-Substring match against name or repoPath.

GET/sessions/{sid}

Locate one session by GUID across the fleet. Returns the same SessionDto shape with machineName / tailnetEndpoint / viewUrl stamped.

curl -s -H "Authorization: Bearer $TOKEN" \
  "http://localhost:7878/sessions/9e0a-..."
GET/directors

List every Director the Gateway currently knows about (HTTP-registered or file-discovered).

curl -s -H "Authorization: Bearer $TOKEN" "http://localhost:7878/directors"
GET/healthz

Liveness probe. Unauthenticated. Returns Director / session counts and the Gateway version.

curl -s "http://localhost:7878/healthz"

Writing to a session (direct-to-Director)

To send a prompt to a session you found via GET /sessions, talk to the owning Director directly. The session payload tells you the URL and the session id, so the client needs only the Director's own bearer token (not the Gateway's). 

curl -s -X POST \
  -H "Authorization: Bearer $DIRECTOR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"text":"How is the rebuild going?"}' \
  "$tailnetEndpoint/sessions/$sessionId/prompt"

Director-side endpoints are documented inline in src/CcDirector.ControlApi/ControlEndpoints.cs. The Gateway intentionally does not duplicate that surface.
