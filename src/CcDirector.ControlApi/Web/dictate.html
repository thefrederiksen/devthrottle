Browser captures audio via Web Audio + AudioWorklet as PCM16 at 24 kHz and streams it over a WebSocket to /dictate. Server transcribes via the OpenAI Realtime API and cleans up via a nano-tier chat model using your dictionary at %LOCALAPPDATA%/cc-director/dictation/dictionary.yaml. If the upstream connection drops, audio is buffered (in memory then on disk) and drained when the connection returns. Timing in the log is seconds since "Start recording" was clicked.
