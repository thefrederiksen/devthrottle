Network addressing

How machines in the fleet address one another. Tailscale (default): each Director is reached over its Tailscale Serve front door. LAN: each Director is reached on its real LAN IP — use only on a trusted network. LAN mode automatically turns on Director authentication (the shared fleet token, gateway.token), so the raw port is never open. Loopback (127.0.0.1) is never used to reach another machine. This is a per-machine setting read at startup; it applies to this host's Directors on their next restart.

Startup

Registers a Run entry under HKCU so the Gateway starts in your session when you log in. The fleet only works while you are logged in, so this is per-user, never a machine service.

Wingman voice agent

Choose which agent and model run as the wingman — the agent that translates a session into a faithful, speakable summary when you use voice mode. Chosen from the agents registered on this machine. These are Gateway settings — they apply fleet-wide, with no per-Director change. Changes take effect after a Gateway restart.

Training data improve the wingman

When on, every wingman voice summary saves up to 20,000 characters of the session terminal plus the wingman's spoken response to wingman-training/ on the Gateway machine — a labeled dataset for testing and improving the wingman. Takes effect immediately, no restart.

Brain hosted agent

The hosted agent process the wingman runs on. It starts on first use; restart replaces a wedged or dead brain or applies a saved agent/model change without a full Gateway restart.
